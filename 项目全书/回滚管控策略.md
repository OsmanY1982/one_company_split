# 回滚管控策略

> 生效日期：2026-06-11 | 实施模块：rollback_control.py

## 三条铁律

| # | 规则 | 说明 |
|---|------|------|
| 1 | **禁止全局回滚** | 不得执行 `git reset --hard`、`git checkout .`、全项目覆盖、批量还原等全局操作 |
| 2 | **逐模块申请** | 一次只回滚一个文件/模块，明确指定路径 |
| 3 | **审计日志** | 所有回滚操作写入 `data/rollback_log.json`，记录时间/模块/原因/操作者 |

## 受保护区域

以下路径在任何情况下禁止回滚：

```
core/              # 核心引擎（cosmic.py / data.py / agent.py 等）
main.py            # 应用入口
gen_book.py        # 文档生成器
rollback_control.py # 回滚管控模块自身
data/              # 数据库目录（独立备份机制）
项目全书/           # 项目文档
源码全书/           # 自动生成文档
```

## AI 回滚流程

```
1. AI 判断需要回滚某模块
      ↓
2. 调用 rollback_control.rollback(module_path, reason)
      ↓
3. 被批准 → 执行回滚 → 更新版本功能地图 → 运行 gen_book.py
   被拒绝 → 向用户说明原因，等待人工介入
```

## 回滚后的文档更新要求

每次回滚完成后，必须在 `项目全书/版本功能地图.md` 追加记录：

```
### YYYY-MM-DD | 回滚：<模块名>
- **回滚模块**：<文件路径>
- **回滚原因**：<原因>
- **回滚记录ID**：RB-xxxx
- **影响说明**：<简要说明>
```

## 审计日志格式

```json
{
  "policy": "module-only",
  "records": [
    {
      "id": "RB-20260611120000",
      "timestamp": "2026-06-11 12:00:00",
      "module": "modules/auth/login_window.py",
      "reason": "WA_TransparentForMouseEvents修复后HUD仍无响应",
      "operator": "AI",
      "status": "approved"
    }
  ]
}
```
