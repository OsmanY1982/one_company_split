## 第十二章 · 订单管理（order_window.py）

### 一个人的公司需要什么订单系统？

大企业的订单系统有审批流、ERP 对接、多仓库、多币种。一个人不需要这些。

一人公司的订单系统只需要：
- 创建订单（客户名、商品、金额、日期）
- 查看所有订单列表
- 按状态过滤（待处理 / 已完成 / 已取消）
- 简单的统计（本月营收、订单数）

### 为什么叫 `order_window` 而不是 `orders_base + orders_window`？

这是项目里反复出现的一个模式：**数据和界面分离**。虽然 order_window.py 本身包含界面和数据操作，但底层的 SQL 操作遵循 base 层设计原则——订单的增删改查都是通过标准的 SQLite 操作，界面只负责渲染和交互。

好处：
1. 数据层可以单独测试（不需要启动 GUI）
2. 未来要换成 Web 端，数据层可以直接复用
3. 所有模块遵守这个模式，新人看代码一目了然
