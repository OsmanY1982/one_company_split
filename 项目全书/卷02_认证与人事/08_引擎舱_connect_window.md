## 第八章 · 引擎舱（connect_window.py）

### 为什么叫引擎舱？

用户登录后面对的第一个界面不是数据看板，而是一个充满科幻感的「引擎舱」。这里有：

- 6 张燃料卡片（Ollama / OpenAI / DeepSeek / Claude / 通义千问 / 自定义）
- 声波解析器（3 档语音引擎选择）
- API Key 输入和配置表单
- Ollama 本地模型扫描
- 点火测试按钮
- 「启动引擎」（带 AI 进舰桥）和「普通操作」（无 AI 进舰桥）两个入口

设计逻辑：就像开车前要检查发动机。你不能让用户在不确认 AI 引擎状态的情况下进入核心操作界面。

### 燃料卡片交互

每张卡片是一个 `EngineGlow` widget——带辉光渐变边框 + 燃料名 + 图标。默认状态：
- 未选中：暗色 + 低透明辉光
- 选中：亮色 + 强辉光脉冲动画
- 不可用：灰色 + 无辉光

卡片下方的配置区域动态切换：选中 Ollama 就显示本地模型列表和扫描按钮，选中 OpenAI 就显示 API Key 输入框。

### Ollama 扫描

用户点击「扫描本星系群」→ 后台线程请求 `http://localhost:11434/api/tags` → 返回已安装的本地模型列表 → 与预设模型合并去重 → 填入 QComboBox。

如果 Ollama 没启动：显示「星门未开启」提示 + 一键启动 Ollama 的按钮（调用 `ollama serve`）。

### 点火测试

点击「点火测试」→ `ConnectionTestThread` 向选中的后端发送 `"ping"` → 等待响应。3 秒内收到回复显示「引擎运转正常 ✓」，超时显示「点火失败」+ 错误信息。

这个功能看起来简单，但非常重要——它让用户在进入核心操作前，确认自己的 AI 配置是正确无误的。

### 状态机

引擎舱有多个状态需要管理：
- 燃料状态：未选 / 已选-未测试 / 已测试-畅通 / 已测试-失败
- 语音状态：未安装 / 安装中（deps）/ 安装中（模型）/ 已安装 / 安装失败
- 配置状态：未填 / 已填-未保存 / 已保存

这些状态全部用 `QPropertyAnimation` + 颜色/文字变化来反馈，而不是弹窗提示。
