## 6. 综合评估与优化建议

### 6.1 高优先级（可立即优化）

1. **Supabase 三项目副本统一**（9 个文件 × 3 副本 = 27 个物理文件）：
   - 将 `activation.py`, `admin_log.py`, `auth.py`, `business.py`, `distribution.py`, `member.py`, `updater.py`, `wallet.py`, `_core.py` 统一为存根 → 删除 management-system 和 planetarium 下的独立副本，改为 `from iqra.core.modules.supabase.xxx import *`

2. **planetarium 与 core 之间的 22 对完全相同副本**：
   - `_ai_shared.py`, `_api_config.py`, `_data.py`, `_engine.py`, `_model_manager_download.py` 等 → 在 planetarium 中改为桥接存根

3. **core/modules/intelligence/backup_p02/** 目录：
   - `agent_bridge.py`（76KB，引用 33 个 iqra.core 模块）→ 标记为备份目录，评估是否可删除

### 6.2 中优先级

4. **`_code_tools.py` 三版本收敛**：
   - 三处分别为 27KB / 11KB / 25KB → 确定规范版本后统一

5. **`_ai_widgets_visualization.py` 收敛**：
   - iqra/modules 5.5KB vs core/modules 20.3KB → 需确认哪个是正确版本

6. **TODO 注释清理**：
   - `cloud_api.py` 和 `event_trigger.py` 在两处各有一份完全相同的 TODO 清单 → 统一模块后只需维护一份
   - `scheduled_tasks.py` 的 "发送通知" TODO 在 4 个项目中各有一份

### 6.3 低优先级

7. **小数差异文件**（< 500B 差异）：
   - `_ai_widgets_business.py`, `_ai_widgets_core.py`, `_compat.py` 等 → 通过 diff 确认差异后统一
